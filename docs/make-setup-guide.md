# Make(Integromat) 시나리오 설정 가이드

> 실버스텝 MVP에서 **운동 완료 → 카카오톡 알림** 자동화를 위한 Make 설정 가이드입니다.

---

## 전체 흐름

```
실버스텝 앱 (운동 완료)
    ↓ HTTP POST (JSON)
Make 웹훅 수신
    ↓
Make 시나리오 실행
    ↓
카카오톡 알림톡 API (비즈엠) 호출
    ↓
자녀 카카오톡에 알림 도착
```

---

## 1단계: Make 계정 가입

1. [make.com](https://www.make.com/) 접속
2. 무료 플랜으로 가입 (월 1,000회 오퍼레이션 무료)
3. 로그인 후 대시보드 진입

---

## 2단계: 새 시나리오 생성

1. 좌측 메뉴 **"Scenarios"** → **"Create a new scenario"** 클릭
2. 첫 번째 모듈로 **"Webhooks"** 검색 → **"Custom webhook"** 선택
3. **"Add"** 클릭 → 웹훅 이름 입력 (예: `SilverStep 운동 완료`)
4. 생성된 **웹훅 URL**을 복사 (예: `https://hook.make.com/abc123...`)

> ⚠️ 이 URL을 실버스텝 앱 설정에서 "웹훅 URL" 필드에 붙여넣기 하세요.

---

## 3단계: 웹훅 데이터 구조 확인

실버스텝이 전송하는 JSON 페이로드:

```json
{
  "seniorName": "홍길동",
  "childName": "홍영희",
  "exerciseName": "만세 운동",
  "exerciseIcon": "🙌",
  "totalReps": 10,
  "duration": "2분 30초",
  "date": "2026년 7월 20일",
  "time": "오전 10:30",
  "timestamp": "2026-07-20T01:30:00.000Z",
  "message": "[실버스텝] 홍길동님이 🙌 만세 운동을 완료했습니다!\n\n📊 횟수: 10회\n⏱️ 시간: 2분 30초\n📅 2026년 7월 20일 오전 10:30\n\n오늘도 건강하게 운동을 마쳤습니다. 안심하세요! 💪"
}
```

### 필드 설명

| 필드 | 설명 | 예시 |
|------|------|------|
| `seniorName` | 어르신 이름 | 홍길동 |
| `childName` | 자녀 이름 | 홍영희 |
| `exerciseName` | 운동 종류 | 만세 운동 |
| `exerciseIcon` | 운동 아이콘 | 🙌 |
| `totalReps` | 완료 횟수 | 10 |
| `duration` | 소요 시간 | 2분 30초 |
| `date` | 날짜 | 2026년 7월 20일 |
| `time` | 시간 | 오전 10:30 |
| `message` | 전체 알림 메시지 (카톡 전송용) | (위 예시 참조) |

---

## 4단계: 카카오톡 알림톡 연결 (비즈엠)

### 옵션 A: 비즈엠 HTTP 모듈 사용 (권장)

1. Make 시나리오에서 웹훅 다음에 **"HTTP" → "Make a request"** 모듈 추가
2. 설정:
   - **URL**: `https://alimtalk-api.bizmsg.kr/v2/sender/send`
   - **Method**: `POST`
   - **Headers**: 
     - `Content-Type`: `application/json`
     - `userId`: (비즈엠 사용자 ID)
   - **Body**:
     ```json
     {
       "message_type": "at",
       "phn": "01012345678",
       "profile": "(비즈엠 프로필 키)",
       "tmplId": "(템플릿 ID)",
       "msg": "{{message}}"
     }
     ```
3. `{{message}}` 부분에 웹훅에서 받은 `message` 필드를 매핑

### 옵션 B: 간단 대안 - Slack/이메일 알림

비즈엠 설정이 어렵다면 테스트 기간 동안 대안 사용:

1. **Slack**: Make에서 "Slack" 모듈 → 특정 채널에 `message` 전송
2. **이메일**: Make에서 "Email" 모듈 → 자녀 이메일로 `message` 전송
3. **텔레그램**: Make에서 "Telegram Bot" 모듈 → 자녀에게 메시지 전송

---

## 5단계: 시나리오 활성화

1. 시나리오 편집 완료 후 하단 **"ON"** 토글로 활성화
2. 스케줄링: **"Immediately"** (즉시 실행) 선택
3. **"Save"** 클릭

---

## 6단계: 테스트

### 실버스텝 앱에서 테스트
1. 앱 시작 화면에서 **⚙️ 설정** 버튼 클릭
2. **웹훅 URL** 필드에 Make 웹훅 URL 붙여넣기
3. **어르신 이름**, **자녀 이름** 입력
4. **"테스트 알림 전송"** 버튼 클릭
5. Make 대시보드에서 시나리오 실행 로그 확인

### Make 대시보드에서 확인
1. **Scenarios** → 해당 시나리오 클릭
2. **"History"** 탭에서 실행 기록 확인
3. 각 모듈의 입출력 데이터 확인 가능

## 트러블슈팅

### 문제: 웹훅이 수신되지 않음
- **확인**: 웹훅 URL이 정확한지 확인 (복사/붙여넣기 시 공백 주의)
- **확인**: Make 시나리오가 **ON** 상태인지 확인
- **확인**: 무료 플랜 오퍼레이션 한도 초과 여부

### 문제: CORS 오류 발생
- Make 웹훅은 기본적으로 CORS를 지원합니다
- 실버스텝 앱에서 자동으로 `no-cors` 모드로 재시도합니다
- `no-cors` 모드에서는 응답을 읽을 수 없지만 전송은 정상 수행됩니다

### 문제: 카카오톡 알림이 오지 않음
- **확인**: 비즈엠 API 키와 템플릿 ID가 올바른지 확인
- **확인**: 수신자 전화번호 형식 (010XXXXXXXX, 하이픈 없이)
- **확인**: 알림톡 템플릿이 승인된 상태인지 확인

### 문제: 시뮬레이션 모드로만 동작
- 앱 설정에서 웹훅 URL이 비어있거나 유효하지 않은 경우 시뮬레이션 모드로 동작합니다
- 유효한 HTTPS URL을 입력하면 실제 전송 모드로 전환됩니다
